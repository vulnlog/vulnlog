== Docker

A container image is published to the GitHub Container Registry on every release.
The image runs as a non-root user (UID 1000) by default.

[source,bash]
----
docker pull ghcr.io/vulnlog/vulnlog:latest
----

All published images are listed at https://github.com/vulnlog/vulnlog/pkgs/container/vulnlog[ghcr.io/vulnlog/vulnlog].

[TIP]
====
Pin to a specific version (for example `ghcr.io/vulnlog/vulnlog:0.12.0`) in CI pipelines to get reproducible builds.
====

=== CI Usage

In most CI environments (GitHub Actions, Jenkins, GitLab CI) the runner's workspace is owned by UID 1000, so the container works without extra flags:

[source,bash]
----
docker run --rm -v "$PWD:/work" ghcr.io/vulnlog/vulnlog:latest validate vulnlog.yaml
----

If your CI runner requires root access, override the user at runtime:

[source,bash]
----
docker run --rm --user root -v "$PWD:/work" ghcr.io/vulnlog/vulnlog:latest suppress vulnlog.yaml
----

=== Local Usage

On most Linux and macOS systems the default user is UID 1000, so bind mounts work out of the box:

[source,bash]
----
docker run --rm -v "$PWD:/work" ghcr.io/vulnlog/vulnlog:latest suppress vulnlog.yaml
----

If your host user has a different UID, pass it explicitly:

[source,bash]
----
docker run --rm -u "$(id -u):$(id -g)" -v "$PWD:/work" ghcr.io/vulnlog/vulnlog:latest suppress vulnlog.yaml
----

Alternatively, use STDIN and STDOUT to avoid writing to the host filesystem from inside the container entirely:

[source,bash]
----
cat vulnlog.yaml | docker run --rm -i ghcr.io/vulnlog/vulnlog:latest suppress --reporter trivy - > .trivyignore.yaml
----

== Native Binary

Standalone native binaries are built with GraalVM and require no Java runtime.
They start instantly and are the recommended choice for local use.

. Download the archive for your operating system and architecture from the https://github.com/vulnlog/vulnlog/releases[releases page].
. Extract the archive.
. Move the `vulnlog` binary to a directory on your `PATH`, for example `/usr/local/bin`.
. Verify the installation:
+
[source,bash]
----
vulnlog --version
----

== JVM Distribution

A platform-independent JVM distribution is also published on the https://github.com/vulnlog/vulnlog/releases[releases page].
It requires Java 21 or later to be installed.

Use the JVM distribution if no native binary is available for your platform.

== Build from Source

Clone the repository and build with Gradle:

[source,bash]
----
git clone https://github.com/vulnlog/vulnlog.git
cd vulnlog
./gradlew installDist
----

The resulting distribution is placed in `build/install/vulnlog`.
Add its `bin` directory to your `PATH` to use it.

[NOTE]
====
Building a native binary from source requires a local GraalVM installation.
See the project's `README.md` for details.
====
