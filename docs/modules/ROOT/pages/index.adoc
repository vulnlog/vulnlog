= Vulnlog

Vulnlog records your team's vulnerability triage decisions in a git-tracked YAML file and turns them into scanner suppression files and shareable reports.
A decision is recorded once: the scanner stops flagging it, the report explains it, and git remembers who made it and when.

New to Vulnlog?
Start with xref:get-started/what-is-vulnlog.adoc[What is Vulnlog?], then xref:get-started/install.adoc[install] the CLI.

== Find your path

You need the impact picture::
Read the report and state how the project is affected, without touching YAML or a build: xref:get-started/quickstart-report.adoc[Quickstart: read the impact report].

Someone asks about a vulnerability::
Answer a customer or user from the recorded decision instead of re-analysing: xref:get-started/quickstart-inquiry.adoc[Quickstart: answer a vulnerability inquiry].
